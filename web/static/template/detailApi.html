1.接口方式

        
所有鲸鱼API采用http连接，以GET/POST的方式进行调用，各项调用参数使用UTF-8编码，响应数据一般为JSON格式。

        
2. API账号&鉴权

        
 用户调用鲸鱼API时需要先申请ApiID，可获得ApiID和相应的ApiSecret。

        
通过调用获取访问凭证API，获取accessToken（访问凭证）和合法访问的时间（默认7200s）

        
调用每个应用API时，在url中统一传入参数accessToken= <accessToken>，则可合法访问API 

        
accessToken过期后，须重新申请

        
3.POST类型API的请求内容提交方式

        
对于提交方式为POST的API，其请求内容统一在Http Body中以JSON格式提交。

        
下文对于API接口请求字段的说明，如无特殊标注，请求样例均为body中的json样式，字段说明均为对json中字段的具体描述。

        
无论get还是post请求，accessToken均在url中传入，而不包含在json中。

        
4.响应结果公共格式

        
鲸鱼API的响应结果均以JSON形式返回，且都为以下标准的结构：

        
字段名称  字段说明

        
success   是否成功返回，取值：true,false

        
code        响应返回码，成功为10200，异常时参考各API的说明

        
 note        提示信息，若success为false时一般为异常信息描述

        
data        具体返回内容，各API会有自己的data结构定义，下文API说明中“响应结果”部分即是对data格式的说明，同时，下文的说明将不再重复描述公共的头部字段。

        
5.响应返回码

        
对于响应返回码，一般遵循以下约定：

        
返回码为5位数字

        
成功码为10200

        
公共异常返回码以2开头

        
各API个性化的返回码以4开头

        
下面列举所有的公共异常返回码：

        
20001    未知错误

        
20002    系统繁忙，稍后再试

        
20003    accessToken无效或已超时

        
20004    不存在的方法

        
20005    缺少参数

        
20006    系统维护中

        
20007    流量限制，稍后再试
